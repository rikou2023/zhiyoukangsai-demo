## 2. 全站视觉基准

### 2.1 色彩令牌

以下色值按用户确认截图的视觉观感整理，落地时可与现有 `tokens.css` 合并：

| 用途 | 建议色值 | 说明 |
|---|---:|---|
| 页面主背景 | `#080F1F` / `#0A1428` | 接近截图里的深海军黑蓝 |
| 区块背景 | `#0F1828` | 比主背景略亮，用于大 section |
| 卡片背景 | `#111B2C` | UI 预览图中的深卡片 |
| 抬起卡片 | `#1A2638` | 用于重点卡片、表单、数据面板 |
| 描边 | `#263246` | 细边框，不要过亮 |
| 强描边 | `#334155` | hover / active / 重点模块 |
| 主按钮蓝 | `#2463FF` / `#2F66FF` | 唯一品牌强调色 |
| 按钮 hover | `#3F7BFF` | 比主蓝略亮 |
| 主文字 | `#F8FAFC` | 大标题、重要文案 |
| 次文字 | `#A8B2C3` | 正文、说明 |
| 弱文字 | `#667085` | 注释、辅助信息 |

### 2.2 禁止项

- 不要紫色、粉色、彩虹渐变。
- 不要玻璃磨砂、透明毛玻璃卡片。
- 不要星空网格、紫粉光晕、漂浮光球。
- 不要过度发光、过度阴影。
- 不要卡通插画，不要泛 AI 抽象插画。
- 不要把首页 Hero 重新画成其它风格。

### 2.3 整体风格

关键词：

```
深色企业级 SaaS / 高端科技 / 极简 B2B / 数据可信 / 系统能力 / 专家服务
```

视觉感觉：

- 像一套成熟企业级数据平台，而不是营销模板。
- 信息密度适中，文字和数据要有呼吸感。
- 图片素材优先使用真实系统截图、证书、合作背书图、流程图。
- 卡片圆角控制在 8px 左右，边框清晰，阴影克制。
- 大标题白色，关键数字和 CTA 用电光蓝。

### 2.4 tryprofound 排版参考

客户明确希望参考 [tryprofound.com](https://www.tryprofound.com/) 的排版。这里的“照抄”只指页面骨架、留白节奏和信息呈现方式，不复制对方品牌、文案、Logo、截图、代码。

需要学习的排版特征：

- 顶部结构：导航 + 公告条 + 居中主叙事。首页 Hero 已确认，不重画；其它页面可以沿用这种“先聚焦一句核心主张”的结构。
- 页面节奏：一屏只讲一个重点，避免把所有内容堆进同一个可视区。
- 内容宽度：正文和标题尽量居中收窄，重点视觉元素再放大到 1100-1280px 宽。
- 产品展示：Hero 后优先出现一个巨大的真实系统截图 / 浏览器窗口 / 产品界面，而不是先堆很多小卡片。
- 分区方式：使用细线框、上下分割线、中心框架、竖向导轨来建立秩序感，不使用复杂装饰背景。
- 卡片密度：少量大卡片优于大量小卡片。不要一排塞 7 个能力卡，改为 4+3 网格、横向分组，或左右交替叙事。
- 视觉主次：每个区块只能有一个主视觉焦点，比如大标题、大数字、系统截图、证书图、流程图。
- 滚动叙事：从痛点、系统能力、团队方案、数据证明、权威背书、最终 CTA 逐步推进，不要像资料目录。

### 2.5 ChatGPT 设计图取舍规则

当前 ChatGPT 设计图只作为视觉参考，不作为文案、业务定位、品牌名称参考。

可以借鉴的 3 个视觉段落：

1. 系统演示的大仪表盘 UI：左侧栏 + 顶部 4 KPI + 中部图表 + 底部明细。这个统一风格的虚拟产品快照可用于重做首页“系统演示”段，比直接摆放风格不统一的真实截图更规整。
2. 6 节点闭环流程图：适合转化为“专属服务”页的交付体系表达，比传统水平时间线更有故事感。落地时仍需服务页保持 4 阶段交付内容，只是视觉上可用闭环节点呈现。
3. 顶栏带下拉指示器的现代化处理：可作为细节优化方向，用在有二级内容的导航项上。

必须保留不动的 7 个现有优势：

1. 首页 Hero 的 AI 平台 Logo 跑马灯，这是客户明确要求。
2. 首页 Hero 的神经网络背景 + 鼠标 180px 节点吸附，这是客户已拍板的核心动效。
3. 客户评价聚光灯轮播：8 条长评 + 5 条短评跑马灯，按客户原话“一个一个轮着放”。
4. 合作品牌 78 个 Logo 双行反向滚动 + 6 大行业客户名单。
5. 产品优势矩阵每列 6 项胶囊，客户原文要求“一模一样照搬”。
6. 行业根基段：4 大数字阵列 + 8 资质徽章（4 专利 / 6 软著 / 100+ 智库 / 30+ 大奖）。
7. 案例数据 countup 入场动画。

文案与品牌硬约束：

- 文字内容 100% 使用客户原文，不能根据设计图改写。
- 公司名只能是“智优康赛 / AIconcepts”，不能使用设计图里的其它公司名。
- 业务定位是 GEO 战略服务商，不是“营销一体化平台”。
- 七大核心能力、产品优势矩阵、权威认证、客户评价、合作品牌全部按客户提供的 docx / `网站文字.md` 一字不改。

## 3. 通用组件规范

### 3.1 顶栏

首页 Hero 已确认，顶栏风格继续沿用：

- 左侧 Logo。
- 中间 7 项导航：首页 / GEO 解决方案 / 客户案例 / 生态资源 / 系统演示 / 关于我们 / 专属服务。
- 当前页使用电蓝半透明 Pill 高亮。
- 右侧按钮：联系我们、预约诊断。
- 顶栏背景可以是深色实底或轻微透明深底，但不要玻璃磨砂感。

### 3.2 按钮

主按钮：

- 背景：电光蓝。
- 文字：白色、半粗体。
- 高度：44-52px。
- 圆角：8px。
- 不要胶囊过圆，不要银白高光按钮。

次按钮：

- 深色背景或透明深底。
- 电蓝描边。
- 文字电蓝或白色。
- hover 时边框和文字略亮。

文字按钮：

- 用于“查看详情 →”。
- 灰蓝文字，hover 变电蓝。

### 3.3 卡片

普通卡片：

- 背景：`#111B2C`。
- 描边：`#263246`。
- 圆角：8px。
- 内边距：32-40px。
- 不要卡片套卡片。

重点卡片：

- 背景：`#1A2638`。
- 描边略亮。
- 可加入顶部 2px 电蓝强调线。

数据卡片：

- 大数字用电光蓝。
- 数字下方放短标签。
- 说明文字灰蓝。
- 数据不要加复杂图标。

### 3.4 表单

表单需要更极简高级：

- 一行或两列输入，避免厚重表单墙。
- 输入框深底，细描边。
- Focus 状态用电蓝描边。
- 表单标题使用：`被 AI 看见，才能被客户选择`。
- 提交按钮使用：`预约专家诊断`。

字段按全站规范：

| 字段 | 类型 |
|---|---|
| 公司名称 | text |
| 姓名 | text |
| 职位 | text |
| 联系电话 | tel |
| 邮箱 | email |
| 核心需求 | textarea |
| 目前最关注的 AI 大模型 | checkbox |
| 目前品牌在 AI 时代的痛点 | radio 或 select |

## 4. 首页设计说明

### 4.1 首页 Hero

状态：已确认，不改。

image2 不需要重新生成首页 Hero。若必须生成整页图，Hero 需要保持现状：

- 居中主标题。
- `第一` 用电光蓝。
- AI 平台 Logo / 名称轮换位在主标题下方。
- 下方是副标题和两个 CTA。
- 背景是神经网络节点，不换成其它背景。

### 4.2 Hero 之后的首页区块

推荐顺序：

| 顺序 | 区块 | 设计重点 |
|---:|---|---|
| 1 | 痛点段 | 大字提问，制造“AI 回答里没有你”的紧张感 |
| 2 | 系统演示 | 真实后台截图大轮播，展示产品可信度 |
| 3 | 七大核心能力 | 7 张能力卡，编号 01-07，卡片规整 |
| 4 | 产品优势矩阵 | 数据 / 分析 / 优化三栏，每栏 6 项 |
| 5 | 案例数据 | 300% / 1.1亿+ / 37.22% 三个大数字 |
| 6 | 权威认证 | 国家高新、专精特新、专利、软著、专家智库 |
| 7 | 客户评价 | 长评聚光灯 + 短评横向展示 |
| 8 | 合作品牌 | Logo 墙 + 行业客户名单 |
| 9 | 留资表单 | 极简高级表单 |

首页 Hero 后续视觉重点：

- 不要继续堆很多同样的深色卡片。
- 区块之间要有明显节奏：大字痛点、真实截图、矩阵、数据、背书、表单。
- 系统截图要比普通卡片更突出。

## 5. 子页设计说明

### 5.1 GEO 解决方案页

页面目标：按客户团队角色讲清楚“你是哪类团队，我们能为你做什么”。

结构：

1. Hero：`依托多年专业技术积累，提供从流量获取到商业转化的一站式解决方案`
2. Lead：完整呈现“无论您的团队负责哪个环节...”段落。
3. 五类团队解决方案：
   - 流量与增长团队
   - 内容与新媒体团队
   - 公关与品牌团队
   - 出海与全球化团队
   - 代理商与生态伙伴
4. 底部统一 CTA / 表单。

设计建议：

- 使用纵向 timeline 或交替左右大卡。
- 每个角色卡包含：编号、团队名称、核心目标、专家护航路径。
- 每张卡右侧可用抽象但克制的数据面板，不要用插画。

### 5.2 客户案例页

页面目标：集中展示商业结果和可信背书。

结构：

1. Hero：`备受全球顶尖营销人员与头部品牌信赖`
2. 三大案例放大卡：
   - 300%
   - 1.1亿+
   - 37.22%
3. 客户长评。
4. 四类客群评价 grid。
5. 短评区。
6. 行业根基与权威认证。
7. 合作品牌 Logo 墙。

设计建议：

- 三大案例要像数据报告首页，数字超大。
- 每个案例卡明确显示“行业 / 核心增长大数 / 服务成效 / 信任资产构建 / 高管原声”。
- 避免评价卡太多导致页面像留言墙。

### 5.3 生态资源页

页面目标：展示全球资源整合能力和战略合作伙伴。

结构：

1. Hero：`汇聚全球顶尖资源，共筑 AI 时代护城河`
2. Google 核心战略伙伴大图卡。
3. SEMrush 中国区唯一代理商大图卡。
4. 全链路出海与 DTC 护航网络。
5. 权威产学研深度协作。
6. 合作伙伴计划 CTA。

设计建议：

- Google / SEMrush 两张图必须放大，不要当小缩略图。
- 这页少用抽象组件，多用真实背书图。
- 版式可以是两张大横卡 + 三列资源网络。

### 5.4 系统演示页

页面目标：让访客亲眼看到系统强大。

结构：

1. Hero：`开启 AI 增长引擎，抢占算法信任红利`
2. 超大系统截图轮播。
3. 三大模块：
   - 预约专家一对一战略演示
   - 获取品牌 AI 搜索深度诊断报告
   - 三大核心智能体驱动能力
4. 三大智能体独立卡片：
   - 需求生成
   - 品牌声誉
   - 多模态内容
5. 留资 CTA。

设计建议：

- 新后台截图是主角，占页面 60-70% 宽。
- 使用深色浏览器窗口容器承载截图。
- 不要把截图裁太狠，必须看清后台细节。

### 5.5 关于我们页

页面目标：建立公司可信度，集中展示硬背书。

结构：

1. Hero：`重塑品牌 AI 认知，交付确定性增长，而非增加您的工作量`
2. 公司介绍 Lead。
3. 核心服务与技术底座 4 项。
4. 证书大图。
5. 出版书籍 4 本。
6. Google / SEMrush 合作背书。
7. 数字阵列：4 / 6 / 100+ / 30+。
8. 资质徽章云。

设计建议：

- 使用真实证书图、书封面、合作图。
- 证书图做横向大图展示。
- 书籍可以做轻微透视陈列，但不要过度 3D。

### 5.6 专属服务页

页面目标：讲清楚服务机制、流程、交付体系。

结构：

1. Hero：`白帽护航，稳占 AI 时代第一推荐位`
2. 白帽 GEO 宣言。
3. 专属服务流程图。
4. 四阶段交付流程：
   - 全网 AI 语境洞察与品牌健康度体检
   - 高价值语料重构与知识库建设
   - 生态共建与模型自然引流
   - 数据监测与效果透明交付
5. 关键能力承诺。
6. 留资 CTA。

设计建议：

- `专属服务服务流程图.png` 必须原图照搬，不可重画。
- 四阶段建议用水平时间线 + 阶段详情卡。
- 强调“白帽、合规、透明、长期”。

### 5.7 需求对接页

页面目标：让用户提交需求，进入专家对接。

结构：

1. Hero：`开启专属增长对话，量身定制您的 AI 破局战略`
2. 三个承诺：
   - 描述核心业务挑战
   - 资深专家一对一响应
   - 即刻获得现状体检 / 竞品洞察 / 破局蓝图
3. 极简表单。

设计建议：

- 表单不要笨重。
- 页面整体可更安静，不要大面积堆卡片。
- 强调“专家直连，不是机器客服”。

## 7. image2 分页 Prompt

## 9. 出图建议

| 批次 | 建议先画 | 原因 |
|---|---|---|
| 第 1 批 | 首页 Hero 后续区块 + 系统演示页 | 最能解决“现在有点丑”的视觉问题 |
| 第 2 批 | 专属服务页 + 关于我们页 | 真实流程图、证书、书籍、合作背书最容易做出可信感 |
| 第 3 批 | 解决方案页 + 客户案例页 + 生态资源页 | 信息密度高，需要前两批确定组件风格后复用 |
| 第 4 批 | 需求对接页 | 表单风格最后统一，不要先做成孤立页面 |

## 10. 设计验收标准

- 是否严格使用用户确认截图的深色 + 电蓝体系。
- 是否保持首页 Hero 不变。
- 是否能看出“企业级 GEO 服务商”，而不是泛 AI 工具站。
- 是否突出真实系统截图、真实背书图、真实流程图。
- 是否减少重复卡片堆叠，形成清晰叙事节奏。
- 是否所有 CTA 都指向统一表单逻辑。
- 是否在移动端也能保持可读，不产生横向溢出。
